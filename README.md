##前端小练习
------

### 1.关于onmousemove事件（01MouseRemove）
  MouseRemove练习主要实现了鼠标移动元素的功能，利用的原理是通过onmousemove事件，获取到移动鼠标的操作对象（鼠标），然后实现box1盒子跟随鼠标移动的效果。在整个js的编写中，最关键的就是获取box1盒子要移动的坐标。这个坐标在通过onmousemove事件返回的对象里，需要从该对象的方法中获取。这个需要考虑浏览器兼容问题，IE8及以下浏览器不支持event，只有window.event，所以要用event = event || window.event来兼容，当文档页面超过浏览器宽高，会有一个scroll滑动值，如果没考虑这个因素，在鼠标移动盒子时会出现问题，因此盒子跟随移动的距离应为鼠标的距离与滑动的距离。

### 2.关于实现切换图片功能（02ChangeImg）
  ChangeImg练习主要实现点击按钮切换图片功能，利用的原理是通过onclick事件。切换照片，主要就是将照片的src属性改变，换为另一张图片的路径。因此，可以将照片的src放入一个数组中存储，当按钮被点击后，将img.src切换为相对应的照片src。获取img标签元素节点时，默认返回一个collection集合，要将它转为一个元素节点，需要在后面加上“[0]”，不然无法对img元素操作。可以初始化一个索引index，通过改变它的值来指向不同的照片，但这个index需要控制范围，不能超出照片长度，不然会有溢出问题。如果超过某个范围，可以将索引指向头尾照片的值。当实现基本的切换图片功能，会发现体验不友好，无法知道当前所显示的照片是第几张，一共有多少张照片，因此可以添加一个p元素，用来显示文本提示用户。使用p元素需要考虑字符串拼接，如果两个值是添加到一起，要加括号绑定，不然会以为是字符串拼接。

### 3.关于实现全选等按钮功能（03AboutCheckBox）
  一共有四个按钮（全选、全不选、反选、提交），六个选择框（5个代表爱好，1个代表全选/不全选状态），实现这个总功能需要考虑五个点：
  > 1.点击全选按钮需要将所有选择框选中
  > 2.点击全不选按钮需要将所有选择框都不选中
  > 3.点击反选按钮需要将当前选中框状态改为未选中，并且5个代表爱好的选择框要与全选选择框状态对应（即只要有1个代表爱好的选择框未选中，全选选择框状态也应为未选中）
  > 4.点击提交按钮需要将当前选中框的内容弹出
  > 5.全选框选中状态要与下面所有选择框对应，并且当操作下面选择框时，若有一个未选中，全选框状态应为未选中，若恢复全选中状态，全选框状态也应该为选中状态
  
  关于checkbox的选中状态，利用了checked属性。当属性值为true，则选择框为选中状态，当为false，选择框为未选中状态
